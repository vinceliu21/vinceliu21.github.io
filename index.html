4U: Personalized Voice Assistant

February 2017

4U is a personalized voice assistant project that was developed during Carnegie Mellon's 2017 Spring Hackathon by myself and three other students. We wanted to leverage the new IBM Watson SDK for Speech to Text (STT) and facial recognition capabilities, therefore we decided that an assistant to help in managing daily activities would be a perfect project.

Key Features

  1. Facial Recognition
    • OpenCV laptop webcam integration to capture user's face
    • IBM Watson Visual Recognition API to train facial recognition model
  2. Restaurant Recommendations
    • Watson STT API to detect "I want to eat ____"
    • Yelp API integration to search local restaurants for the desired food
  3. Budget Planning
    • Watson STT API to allow for user to log budget activity through speech

Zokyo: GitHub Chrome Extension

January 2019

Zokyo was a project to learn the process and intricacies of developing chrome extensions. I built a chrome extension that helped developers manage their GitHub tickets. Developers using Zokyo could view, edit, and close their GitHub tickets from any chrome tab.

I launched the project on the Chrome Developer Store and posted on Product Hunt where I was on the front page for that day's product launches. The project is not maintained anymore, but you can find more information here.

CloudTrack: AWS/GCP Slackbot

2019

CloudTrack is a Slackbot built with Amazon Web Services (AWS) and Google Cloud Platform (GCP) integration that posts daily updates about currently live cloud instances and their specifics: type of instance, instance name, and instance zone.

The inspiration for CloudTrack came from the too often occurrence of deploying cloud instances during development and then not shutting them down properly afterwards. It is also easy for developers to quickly forget about how many instances they have deployed. Developers use a lot of services, but developers are always working in Slack and so that is what led to the inspiration for a Slackbot interface.

Build18: Holographic Arcade Table

2016

For Carnegie Mellon's 2016 hardware Hackathon, I partnered with three other students to build an arcade table where two users could control holograms and battle one another. The holograms were displayed using an acrylic pyramid, a monitor, and pepper's ghost for the illusion effect. The arcade controls were built using a Raspberry Pi and Arduino Mega, while the game logic was made with Python.

We were awarded the Build 18 Officer's Choice award for the judge's favorite project out of the 80 other projects.
